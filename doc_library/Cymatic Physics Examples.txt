### 3. Cymatic "Relaxation" as a Procedural Operator
Houdini uses "Relax" or "Smooth" SOPs to clean up procedural geometry. You use the **Cymatic Solver.**
*   **The Optimization:** Instead of complex boolean math (SVDAG subtraction), use **High-Viscosity Cymatic Settlement.**
*   **Example:** When generating a "Procedural Cave" junction:
    1.  Rough-cut the SVDAG using overlapping box-spheres.
    2.  Run the Cymatic solver for 10 frames with `substrate_viscosity = 1.0` (Mud-like).
    3.  The "Liquid" density field will naturally smooth the sharp edges into organic arches.
    4.  **Finalize:** Flip the resulting density back into SVDAG `1`s and `0`s.

### 4. "Symmetry-Protected" Delta Buffering
In Houdini, modifying a model often requires re-calculating the whole node graph.
*   **The Optimization:** Use **Copy-on-Write (CoW) SVDAG branches.** 
*   Your `ContentRect` points to a `svdag_root_index`. When you "procedurally" damage or modify it, you only create a new branch for the **delta**. 
*   **The "Inheritance" Logic:** The `ContentRect` continues to reference the "Global Pattern" for untouched areas and only looks at the `Thread-Local Delta Pool` for the procedural additions.

### 5. Epoch-Gated Procedural Generation
Proceduralism usually "ticks" every frame. You must gate it with your **Epoch System.**
*   **The Optimization:** Treat procedural generation as a **Radiance Probe for Geometry.**
*   Generate the "Procedural Details" (like moss growth or grime) only when the `geometry_epoch` or `light_epoch` suggests a state change. 
*   **Example:** A "Procedural Door" only re-evaluates its "Rust Patterns" if the `moisture` field (Cymatic property) has changed since the last `last_cymatic_epoch`.

### 6. Zig 0.15.1 "Silo" Memory Pinning (The SIMD Optimization)
For Houdini-style mass-scale proceduralism (e.g., generating 1,000 trees):
*   **Batching by Material:** Use your `WorkDistribution` to group all procedural boxes that share the same **PHE Kernel (Material ID)**.
*   **Pre-Fetch Buffer:** In the thread's `scratch_memory_ptr`, pre-calculate the **Perlin Noise Lattice** for the entire `WorkBatch`. 
*   **Execution:** The `shadeHit` function then becomes a simple lookup into the thread-local noise buffer rather than calculating `evaluatePerlin` 16 million times per frame.

### 7. Super-AABB Cull for Procedural Logic
Don't run procedural logic on boxes that aren't on screen.
*   **The Optimization:** Anchor the "Procedural Generator" to the **Super AABB**.
*   The `FrameCoordinator` only invokes the "Prolog Assembly" phase if the `Super AABB` of the root `ContentRect` intersects the Frustum. 
*   **The "Lazy Build":** The building exists as a 40-byte `Fact` until the player looks at it; only then does it "unfold" into a `rects[]` hierarchy in the thread's local arena.
