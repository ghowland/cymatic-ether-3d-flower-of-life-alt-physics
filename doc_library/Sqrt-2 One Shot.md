# Deriving √2 with Integer Tuples: The VFR Method

## Premise

Real numbers and rational numbers are fundamentally different systems. Rationals are finite, exact, and support equality. Reals (in practice) require infinite processes, lose equality, and accumulate drift under sequential operations. These are not the same kind of mathematics, and should not be treated as interchangeable.

The question: can we describe √2 — the canonical irrational number — entirely within integer arithmetic, without losing any information?

The answer is yes.

---

## Step 1: The Problem as Geometry

Place a square on a pixel grid. A 200×200 box. We want the diagonal.

The diagonal is a line from (0,0) to (200,200). Its slope is 200/200 = 1/1. This is a ratio of two integers. On the grid, we walk it: right 1, up 1, repeated 200 times. Every coordinate is an integer. The walk is exact.

The diagonal *length* is where √2 enters: the length is 200√2. But we did not need the length to *describe* or *walk* the diagonal. We needed only the rise and run: two integers.

This is the key observation: the geometry that produces √2 is fully specified by integers. The irrational number appears only when we ask a particular *derived* question (length), not when we describe the object itself.

---

## Step 2: The Pell Relation

For any integers V and F, compute:

    R = V² − 2F²

R is always an integer. This is the Pell relation for √2.

If R = 0, then V/F = √2 exactly, meaning √2 is rational. But √2 is irrational (Euclid's proof). Therefore R = 0 has no nonzero integer solution. R is never zero.

Since R is an integer and can never be zero, the tightest it can get is |R| = 1. And indeed, there are infinitely many integer pairs (V, F) where R = ±1 exactly. These are the Pell solutions.

---

## Step 3: The VFR Tuple

Define a tuple [V, F, R] where:

- V and F are positive integers
- R = V² − 2F² = ±1

This is not an approximation. It is an exact identity over the integers. The tuple [7, 5, −1] means:

    7² − 2(5²) = 49 − 50 = −1

This is a fact. It is exact. It is finite. It is verifiable by integer multiplication.

The ratio V/F = 7/5 = 1.4 is a rational number near √2. The remainder R = −1 tells us *exactly* which side: since R < 0, we know V/F < √2. The tuple encodes both the approximation and its exact relationship to √2, entirely in integers.

---

## Step 4: The Recursion

The Pell solutions are generated by a simple integer recurrence.

Seed: (V₀, F₀) = (1, 1)

Rule:

    V' = V + 2F
    F' = V + F

Applying this:

    [1,  1,  −1]  →  1² − 2(1²)   = −1
    [3,  2,  +1]  →  3² − 2(2²)   = +1
    [7,  5,  −1]  →  7² − 2(5²)   = −1
    [17, 12, +1]  →  17² − 2(12²)  = +1
    [41, 29, −1]  →  41² − 2(29²)  = −1
    [99, 70, +1]  →  99² − 2(70²)  = +1

The remainder alternates: −1, +1, −1, +1, ... It never reaches zero. It never will. This alternation is the exact arithmetic signature of the irrationality of √2, expressed entirely in ℤ.

---

## Step 5: What the Remainder Means

For any VFR tuple [V, F, R]:

    (V/F)² = 2 + R/F²

When R = −1: (V/F)² = 2 − 1/F², so V/F < √2. The ratio undershoots.

When R = +1: (V/F)² = 2 + 1/F², so V/F > √2. The ratio overshoots.

The distance from 2 in the squared metric is exactly 1/F², which shrinks as F grows. But R itself stays at ±1 forever. The approximation gets tighter, but the remainder never vanishes — because vanishing would require √2 to be rational.

---

## Step 6: The Bilateral Remainder Notation

Since the remainder always alternates ±1 for √2, we can write this as a single notation:

    [V, F, ±1]

or in text: [7, 5, +-1]

This means: the remainder is not a single value but a defined behavior — it alternates between −1 and +1 on each evaluation. This is not ambiguity. It is the complete and exact description of how an integer ratio relates to an irrational target.

The ±1 is not error. It is not drift. It is a component of the value, analogous to how (a, b) represents a + bi in complex numbers. The bilateral remainder is part of the number.

---

## Step 7: Walking the Grid

Return to the 200×200 box. The diagonal is walked with [1, 1, −1]: rise 1, run 1, remainder −1.

Take 200 steps. At each step, the Pell relation holds: 1² − 2(1²) = −1. The remainder alternates sign with each step: −1, +1, −1, +1, ...

After 200 steps (even number): 100 pairs of (−1, +1). Each pair sums to zero. Total remainder: 0.

The walk arrives at (200, 200) with exact coordinates and zero accumulated remainder. No reals were used. No floats. No truncation. No drift. The result is exact in ℤ.

For 201 steps (odd number): 100 pairs cancel, one −1 remains. Total remainder: −1. Still exact. Still integer. The remainder is known, not lost.

---

## Step 8: The General Principle

For any N×N grid walked with [1, 1, ±1]:

- N even → total remainder = 0, exact closure
- N odd → total remainder = ±1, exact and known

For non-square rectangles, higher Pell convergents ([7, 5, ±1], [17, 12, ±1], etc.) provide the rise-run ratio at whatever precision the grid demands.

In every case:

- All quantities are integers
- All operations are integer arithmetic
- The remainder is exact at every step
- Equality is preserved throughout
- No real number arithmetic is ever performed

---

## Summary

√2 cannot be written as a single ratio V/F. The VFR framework does not claim otherwise.

What VFR provides is an exact, finite, integer encoding of the relationship between rational approximants and √2. The tuple [V, F, ±1] where V² − 2F² = ±1 captures:

1. The rational approximant (V/F)
2. Which side of √2 it falls on (sign of R)
3. The exact distance in the squared metric (1/F²)
4. The irrationality itself (R ≠ 0 permanently, alternates ±1)

All of this lives in ℤ. No component is irrational. No component is infinite. No component is approximate. The bilateral remainder ±1 is the irrationality of √2, expressed as an integer property.

This is not a replacement for ℝ. It is a recognition that ℝ and ℚ are different systems, and that many problems — particularly those on discrete grids — are naturally and completely solved in ℚ (and ℤ) without invoking ℝ at all.

The notation encodes what the integers actually do. Nothing more. Nothing less.
